¿Qué es Git?

Es un sistema de control de versiones de código abierto y gratuito que se utiliza para realizar un seguimiento de los cambios en los archivos y carpetas de un proyecto de software a lo largo del tiempo. Es una herramienta de colaboración esencial para los desarrolladores de software y otros profesionales de la informática, ya que permite a los usuarios trabajar juntos en el mismo código y mantener un historial completo y detallado de los cambios realizados en el proyecto.

Con Git, los desarrolladores pueden realizar un seguimiento de los cambios en el código fuente, lo que les permite volver a versiones anteriores del proyecto en caso de errores o problemas. También pueden trabajar en diferentes ramas de un proyecto, lo que les permite experimentar con nuevas funciones y características sin afectar el código principal.

Además, facilita la colaboración entre diferentes usuarios, ya que permite a los desarrolladores trabajar en sus propias copias del proyecto y luego fusionar los cambios en una sola versión del código. Esto hace que el trabajo en equipo sea más eficiente y reduce la posibilidad de conflictos y errores en el proyecto.

Instalación de Git

Windows:

  • Descarga el instalador de Git desde la página web oficial: https://git-scm.com/download/win
  • Ejecuta el archivo descargado y sigue las instrucciones del asistente de instalación.
  • Asegúrate de seleccionar la opción "Agregar Git al PATH de Windows" durante la instalación, para que puedas ejecutar Git desde la línea de comandos.

Una vez instalado Git en Windows, puedes abrir la línea de comandos (cmd) y verificar que Git está instalado y funcionando correctamente con el comando:

          git --version

macOS:

  • Abre la terminal de macOS.
  • Instala Homebrew, un gestor de paquetes para macOS, con el siguiente comando en la terminal:
          /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Instala Git con el siguiente comando:

          brew install git

Una vez instalado Git en macOS, puedes verificar que está instalado y funcionando correctamente con el comando:

          git --version

Configuración básica:

Después de instalar Git, es importante configurar tu nombre y dirección de correo electrónico. Esto se debe a que Git utiliza esta información para registrar quién realiza los cambios en el código.

Para configurar tu nombre y dirección de correo electrónico, abre una línea de comandos y ejecuta los siguientes comandos, reemplazando "Tu nombre" y "tu-correo@example.com" con tus propios datos:

        git config --global user.name "Tu nombre" 
        git config --global user.email "tu-correo@example.com"

Configuración avanzada:

Git tiene muchas opciones de configuración avanzada para ajustar su comportamiento a tus necesidades. Puedes ver todas las opciones disponibles con el comando:

          git config --list

Algunas de las opciones más comunes incluyen:

  • core.editor: te permite configurar el editor de texto que Git utiliza para abrir y editar archivos de texto.
  • core.autocrlf: te permite configurar cómo Git maneja los saltos de línea de los archivos. Esta opción es importante si trabajas en un equipo que utiliza diferentes sistemas operativos.
  • color.ui: te permite configurar si Git muestra salida en color o no.
Puedes configurar estas opciones usando el siguiente comando:
          git config --global (nombre-de-la-opción) (valor-de-la-opción)

Por ejemplo, para configurar el editor de texto Vim como el editor predeterminado de Git, puedes ejecutar el siguiente comando: 

          git  config --global core.editor vim

Creación de un repositorio

Un repositorio de Git es un directorio donde se almacenan los archivos del proyecto, junto con todo el historial de cambios y las ramas de desarrollo.

Creación de un repositorio local:

  • Abre una línea de comandos y navega hasta el directorio raíz de tu proyecto.
  • Inicializa un nuevo repositorio de Git en el directorio con el siguiente comando:
    •           git init
  • Ahora Git está listo para rastrear los cambios en los archivos de tu proyecto. Puedes agregar archivos al repositorio con el comando:
    •           git add (nombre-de-archivo)
  • Después de agregar los archivos, puedes confirmar los cambios con el comando:
    •           git commit -m "Mensaje de confirmación"

Creación de un repositorio remoto:

Un repositorio remoto es un repositorio alojado en un servidor en línea (como GitHub, GitLab o Bitbucket). Puedes usar un repositorio remoto para colaborar con otros desarrolladores y para hacer copias de seguridad de tu código.

Para crear un repositorio remoto en GitHub, por ejemplo, sigue los siguientes pasos:

  • Crea una cuenta en GitHub (si aún no tienes una).
  • Inicia sesión en tu cuenta de GitHub y haz clic en el botón "New" para crear un nuevo repositorio.
  • Asigna un nombre al repositorio y selecciona las opciones de configuración deseadas.
  • Haz clic en "Create repository" para crear el repositorio en GitHub.

Después de crear el repositorio remoto en GitHub, puedes vincular tu repositorio local a él con los siguientes comandos:

        git remote add origin (URL-del-repositorio-remoto)
          git push -u origin master

Reemplaza (URL-del-repositorio-remoto) con la URL de tu repositorio remoto en GitHub.

El ciclo básico de trabajo en Git

  • Agregar cambios: El primer paso es agregar los cambios realizados en los archivos a un área temporal llamada "área de preparación" o "index". Esto se hace con el comando git add. Por ejemplo, si quieres agregar todos los cambios realizados en los archivos de tu directorio actual, puedes usar el comando git add .
  • Confirmar cambios: El segundo paso es confirmar los cambios agregados en el área de preparación con el comando git commit. Debes proporcionar un mensaje que describa brevemente los cambios realizados. Por ejemplo, si has agregado un archivo nuevo llamado "archivo.txt" y has hecho algunos cambios en "archivo2.txt", puedes confirmar los cambios con el comando git commit -m "Agregado archivo.txt y cambios en archivo2.txt".
  • Enviar cambios: El tercer paso es enviar los cambios confirmados a un repositorio remoto con el comando git push. Debes especificar el nombre del repositorio remoto y la rama a la que deseas enviar los cambios. Por ejemplo, si el repositorio remoto se llama "origin" y deseas enviar los cambios a la rama "main", puedes usar el comando git push origin main.

Ramificación y fusión (branching y merging)

Ramificación (Branching):

La ramificación es el proceso de crear una nueva línea de desarrollo en un repositorio de Git. Cada rama representa una versión diferente del código que se puede modificar sin afectar el código en otras ramas.

Para crear una nueva rama en Git, ejecuta el siguiente comando:

          git branch (nombre-de-la-rama)

Para cambiar a la nueva rama que acabas de crear, ejecuta el siguiente comando: 

          git checkout (nombre-de-la-rama)

O puedes hacer ambos pasos de forma abreviada con el comando:

          git checkout -b (nombre-de-la-rama)

Una vez que estás trabajando en una rama, puedes hacer todos los cambios que necesites en los archivos de tu proyecto sin afectar el código en otras ramas.

Fusión (Merging):

La fusión es el proceso de combinar dos o más ramas en un repositorio de Git. Cuando hayas terminado de trabajar en una rama y quieras integrar tus cambios en la rama principal (también conocida como la rama "master"), debes fusionar las dos ramas.

Para fusionar dos ramas en Git, sigue los siguientes pasos:
  • Cambia a la rama en la que deseas integrar los cambios:
    • git checkout (rama-a-fusionar-en)
  • Ejecuta el comando de fusión con el nombre de la rama que deseas fusionar:
    •             git merge (nombre-de-la-rama-que-quieres-fusionar)

En algunos casos, puede haber conflictos durante el proceso de fusión. Esto sucede cuando Git no puede determinar automáticamente cómo combinar los cambios de dos ramas. En esos casos, Git te pedirá que resuelvas los conflictos manualmente.

Para resolver un conflicto de fusión en Git, abre el archivo en conflicto en tu editor de texto y resuelve las diferencias manualmente. Una vez que hayas resuelto todos los conflictos, haz un commit para finalizar la fusión.

Trabajar con repositorios remotos

Conexión con un repositorio remoto:

Para conectar tu repositorio local a un repositorio remoto existente, debes agregar una URL remota al repositorio local. Puedes hacerlo ejecutando el siguiente comando en tu consola:

        git remote add (nombre-remoto) (URL-del-repositorio-remoto)

Por ejemplo, si deseas agregar un repositorio remoto llamado "origin" a tu repositorio local, ejecuta el siguiente comando:

        git remote add origin https://github.com/usuario/repositorio.git

Sincronización de cambios:

Para sincronizar cambios entre tu repositorio local y el repositorio remoto, puedes usar los siguientes comandos:

  • git pull: este comando descarga los cambios del repositorio remoto y los fusiona automáticamente con tu rama actual en el repositorio local.
  • git push: este comando envía los cambios de tu repositorio local al repositorio remoto.

Por ejemplo, si deseas descargar los cambios del repositorio remoto y fusionarlos con tu rama actual en el repositorio local, ejecuta el siguiente comando:

        git pull (nombre-remoto) (nombre-rama)

Si deseas enviar los cambios de tu repositorio local al repositorio remoto, ejecuta el siguiente comando:

        git push (nombre-remoto) (nombre-rama)

Es importante tener en cuenta que puede haber conflictos al sincronizar cambios entre el repositorio local y el remoto, especialmente si múltiples desarrolladores están trabajando en el mismo archivo o línea de código. En este caso, Git te pedirá que resuelvas manualmente los conflictos antes de continuar con la sincronización.

Etiquetas (tags)

Las tags en Git son punteros a puntos específicos en el historial del repositorio. Se usan a menudo para marcar versiones de un proyecto.

Crear una etiqueta:

Para crear una etiqueta en Git, puedes usar el comando git tag. Hay dos tipos de etiquetas: ligera (lightweight) y anotada (annotated).

  • Etiqueta ligera: es simplemente un puntero a un commit específico en el historial de Git. Para crear una etiqueta ligera, ejecuta el siguiente comando:
    •           git tag (nombre-etiqueta) (commit)
  • Por ejemplo, si deseas crear una etiqueta ligera llamada "v1.0" para el commit con el ID abc123, ejecuta el siguiente comando:
    •           git tag  v1.0 abc123
  • Etiqueta anotada: es un objeto Git completo que incluye información adicional, como el nombre del etiquetador, la fecha y un mensaje. Para crear una etiqueta anotada, ejecuta el siguiente comando:
    •           git tag -a (nombre-etiqueta) (commit) -m "(mensaje)"
  • Por ejemplo, si deseas crear una etiqueta anotada llamada "v1.0" para el commit con el ID abc123 con un mensaje de "Versión 1.0", ejecuta el siguiente comando:
    •           git tag -a v1.0 abc123 -m "Versión 1.0"

Ver etiquetas:

Para ver las etiquetas existentes en el repositorio, ejecuta el siguiente comando:

        git tag

Esto mostrará una lista de todas las etiquetas en orden alfabético.

Si deseas ver la información detallada de una etiqueta anotada en particular, ejecuta el siguiente comando:

        git show (nombre-etiqueta)

Eliminar una etiqueta:

Para eliminar una etiqueta existente en Git, puedes usar el comando git tag con la opción -d.

Por ejemplo, si deseas eliminar una etiqueta llamada "v1.0", ejecuta el siguiente comando:

        git tag -d v1.0

Es importante tener en cuenta que eliminar una etiqueta no afecta al historial de Git. Si una etiqueta se ha eliminado por error, puedes recrearla fácilmente.

Herramientas y recursos útiles de Git

  • GitKraken: una herramienta gráfica para Git que proporciona una interfaz visual intuitiva para trabajar con repositorios, ramas, fusiones, etc.
  • GitHub: un servicio de alojamiento de repositorios Git en la nube que permite a los desarrolladores colaborar y compartir su código.
  • GitLab: una alternativa de GitHub que también proporciona alojamiento de repositorios Git en la nube y herramientas de colaboración.
  • Bitbucket: otra alternativa de GitHub que ofrece alojamiento de repositorios Git en la nube y herramientas de colaboración.
  • Git Bash: una terminal que permite a los usuarios de Windows utilizar la línea de comandos de Git como si estuvieran en un entorno Unix.
  • Documentación oficial de Git: la documentación oficial de Git proporciona una guía completa para utilizar Git, incluyendo comandos, conceptos, flujos de trabajo, etc.
