Introducción a SQL

Publicado el día 20 de marzo, 2023 a las 12:00 PM

¿Qué es SQL y para qué se utiliza?

SQL se utiliza para realizar diversas tareas relacionadas con el manejo de datos, como la creación, modificación y eliminación de tablas, la inserción, actualización y eliminación de registros, la consulta y extracción de datos de la base de datos, la gestión de permisos y seguridad, entre otras. La mayoría de los sistemas de gestión de bases de datos relacionales, como MySQL, Oracle, PostgreSQL, SQL Server, entre otros, utilizan SQL como lenguaje estándar para interactuar con sus bases de datos.

En resumen, SQL es un lenguaje de programación fundamental para el manejo de datos en sistemas de bases de datos relacionales y su conocimiento es esencial para cualquier profesional que trabaje con datos en la actualidad.

¿Qué son las bases de datos?

Las bases de datos se utilizan para almacenar información de forma estructurada, lo que permite realizar búsquedas, consultas, modificaciones y eliminaciones de manera eficiente y rápida. Además, las bases de datos pueden almacenar grandes cantidades de datos de manera segura y controlada. Existen diferentes tipos de bases de datos, pero las más comunes son las bases de datos relacionales y las bases de datos NoSQL.

Existen diferentes tipos de bases de datos

Las más comunes son las bases de datos relacionales y las bases de datos NoSQL.

  • Las bases de datos relacionales utilizan tablas para almacenar la información de forma estructurada, lo que permite establecer relaciones entre las diferentes tablas y realizar consultas complejas utilizando lenguajes de programación como SQL
  • Las bases de datos NoSQL, por otro lado, no utilizan tablas y se basan en modelos de datos diferentes a los utilizados en las bases de datos relacionales. Estos modelos pueden incluir documentos, gráficos o clave-valor, entre otros.

En resumen, una base de datos es un sistema organizado de almacenamiento de datos que se utiliza para almacenar, recuperar y gestionar información de forma eficiente y segura. Las bases de datos son fundamentales para el funcionamiento de muchas aplicaciones y sistemas informáticos en la actualidad.

Estructuras de las BBDD

Las bases de datos relacionales se estructuran en tablas, que se componen de filas y columnas. Cada tabla representa una entidad o concepto en la base de datos y contiene información relacionada con ese concepto.

Cada columna de la tabla representa un atributo o característica de la entidad y contiene datos de un tipo específico, como texto, números, fechas, entre otros. Cada fila de la tabla representa una instancia de la entidad y contiene valores para cada uno de los atributos de la entidad.

Por ejemplo, si tuviéramos una tabla "Clientes" en una base de datos de una empresa, podríamos tener las siguientes columnas: "ID del cliente" , "Nombre" , "Apellido" , "Dirección" , "Correo electrónico" , "Teléfono" , entre otras. Cada fila de la tabla representaría un cliente en particular y tendría valores para cada una de las columnas correspondientes.

Las bases de datos NoSQL pueden tener estructuras diferentes, dependiendo del modelo de datos utilizado. Por ejemplo, en una base de datos de documentos, los datos se almacenan en documentos que pueden tener una estructura flexible y anidada, similar a los objetos en la programación orientada a objetos.

Tipos de datos en SQL

En SQL hay diferentes tipos de datos que se utilizan para definir los atributos de las tablas y los valores que pueden almacenar. A continuación, te explico los tipos de datos más comunes en SQL:

  • Datos numéricos: se utilizan para almacenar valores numéricos y pueden ser enteros o con decimales. Los tipos de datos numéricos más comunes son INTEGER, BIGINT, SMALLINT, NUMERIC, DECIMAL y FLOAT.
  • Datos de caracteres: se utilizan para almacenar valores de texto y pueden ser de longitud fija o variable. Los tipos de datos de caracteres más comunes son CHAR, VARCHAR y TEXT.
  • Datos de fecha y hora: se utilizan para almacenar fechas y/o horas y pueden ser de diferentes formatos. Los tipos de datos de fecha y hora más comunes son DATE, TIME, DATETIME y TIMESTAMP.
  • Datos booleanos: se utilizan para almacenar valores verdadero o falso (TRUE o FALSE) y se representan con el tipo de datos BOOLEAN.
  • Datos binarios: se utilizan para almacenar datos binarios, como imágenes o archivos, y se representan con el tipo de datos BLOB o BYTEA.
  • Datos de enumeración: se utilizan para limitar los valores que se pueden almacenar a un conjunto específico de opciones y se representan con el tipo de datos ENUM.

Estos son solo algunos de los tipos de datos más comunes en SQL. Dependiendo del sistema de gestión de bases de datos y del lenguaje de programación utilizado, pueden haber más tipos de datos disponibles. Es importante tener en cuenta el tipo de dato adecuado para cada atributo de la tabla para asegurar la integridad de los datos y evitar errores de inserción o consulta.

Consultas básicas en SQL

Supongamos que tenemos una tabla llamada "Clientes" que tiene las siguientes columnas: "ID del cliente" , "Nombre" , "Apellido" , "Edad" , "Dirección" y "Correo electrónico"

Ejemplo de SELECT:

La cláusula SELECT se utiliza para especificar las columnas que queremos recuperar de la tabla. Por ejemplo, para recuperar el nombre y apellido de todos los clientes, podríamos escribir la siguiente consulta SQL:

            SELECT Nombre , Apellido
            FROM Clientes ;

Esta consulta devolvería una lista de todos los nombres y apellidos de los clientes en la tabla "Clientes".

Ejemplo de FROM:

La cláusula FROM se utiliza para especificar la tabla de la que queremos recuperar los datos. Por ejemplo, para recuperar la dirección de un cliente en particular, podríamos escribir la siguiente consulta SQL:

            SELECT Dirección
            FROM Clientes
            WHERE ID del cliente = 12;

Esta consulta devolvería la dirección del cliente con el ID 12.

Ejemplo de WHERE:

La cláusula WHERE se utiliza para filtrar los datos de la tabla. Por ejemplo, para recuperar todos los clientes que tengan menos de 30 años, podríamos escribir la siguiente consulta SQL:

          SELECT Nombre, Apellido, Edad
          FROM Clientes
          WHERE Edad < 30 ;

Esta consulta devolvería una lista de todos los clientes que tienen menos de 30 años, junto con sus nombres y edades.

Funciones de agregación:

Las funciones de agregación se utilizan para realizar cálculos sobre los datos de una tabla. Algunas de las funciones de agregación más comunes en SQL son: SUM, AVG, COUNT, MAX y MIN. Por ejemplo, si queremos calcular la suma total de los precios de todos los productos en una tabla "Productos", podríamos utilizar la función SUM de la siguiente manera:

            SELECT SUM(Precio)
            FROM Productos;

Operadores lógicos:

Los operadores lógicos se utilizan para combinar condiciones en una consulta SQL. Los operadores más comunes son: AND, OR y NOT. Por ejemplo, si queremos recuperar los clientes que tienen más de 25 años y viven en Buenos Aires, podríamos utilizar el operador AND de la siguiente manera:

            SELECT *
            FROM Clientes
            WHERE Edad > 25 AND Cuidad = 'Buenos Aires' ;

Operadores aritméticos:

Los operadores aritméticos se utilizan para realizar operaciones matemáticas en una consulta SQL. Los operadores más comunes son: + (suma), - (resta), * (multiplicación) y / (división). Por ejemplo, si queremos calcular el precio total de un pedido que contiene 5 productos de un precio unitario de 10 dólares, podríamos utilizar el operador * de la siguiente manera:

            SELECT 5 * 10 AS 'Precio Total';

En este ejemplo, la consulta SQL devolvería el valor 50 como resultado del cálculo realizado mediante el operador *.

Ordenamiento y agrupamiento

ORDER BY:

La cláusula ORDER BY se utiliza para ordenar los resultados de una consulta SQL por una o varias columnas. Por ejemplo, si queremos mostrar los nombres de todos los clientes en orden alfabético, podríamos escribir la siguiente consulta SQL:

            SELECT Nombre
            FROM Clientes
            ORDER BY Nombre ASC ;

En este ejemplo, la consulta SQL ordenará los nombres de los clientes en orden ascendente (de la A a la Z) y devolverá los resultados en ese orden.

GROUP BY:

La cláusula GROUP BY se utiliza para agrupar los resultados de una consulta SQL por una o varias columnas. Por ejemplo, si queremos contar cuántos clientes hay en cada ciudad, podríamos escribir la siguiente consulta SQL:

            SELECT Ciudad, COUNT(*) AS 'Numero de clientes'
            FROM Clientes
            GROUP BY Cuidad ;

En este ejemplo, la consulta SQL agrupará los clientes por ciudad y contará el número de clientes en cada ciudad. Los resultados se mostrarán en una tabla que tendrá dos columnas: "Ciudad" y "Número de clientes".

Modificación de datos en SQL

INSERT

La cláusula INSERT se utiliza para insertar nuevos registros en una tabla. Por ejemplo, si queremos agregar un nuevo producto a una tabla "Productos", podríamos utilizar la siguiente consulta SQL:

           INSERT INTO Productos (Nombre, Precio, Stock)
           VALUES ('Nuevo Producto', 50.00, 100);

En este ejemplo, la consulta SQL insertará un nuevo registro en la tabla "Productos" con los valores 'Nuevo Producto', 50.00 y 100 en las columnas "Nombre" , "Precio" y "Stock" , respectivamente.

UPDATE

La cláusula UPDATE se utiliza para actualizar los registros existentes en una tabla. Por ejemplo, si queremos actualizar el precio de un producto en la tabla "Productos" , podríamos utilizar la siguiente consulta SQL:

           UPDATE Productos
           SET Precio = 60.00
           WHERE Nombre = 'Nuevo Producto';

En este ejemplo, la consulta SQL actualizará el precio del producto con nombre 'Nuevo Producto' a 60.00 en la tabla "Productos" .

DELETE

La cláusula DELETE se utiliza para eliminar registros de una tabla. Por ejemplo, si queremos eliminar un producto de la tabla "Productos" , podríamos utilizar la siguiente consulta SQL:

           DELETE FROM Productos
           WHERE Nombre = 'Nuevo Producto';

En este ejemplo, la consulta SQL eliminará el registro del producto con nombre 'Nuevo Producto' de la tabla "Productos" .

Unión de tablas en SQL

Supongamos que tenemos dos tablas: "Clientes" y "Pedidos" . La tabla "Clientes" contiene información sobre los clientes, mientras que la tabla "Pedidos" contiene información sobre los pedidos realizados por los clientes. Ambas tablas comparten una columna "IDCliente" que se utiliza para relacionar los registros de ambas tablas.

INNER JOIN:

La cláusula INNER JOIN se utiliza para mostrar solo los registros que tienen una coincidencia en ambas tablas. Por ejemplo, si queremos mostrar los nombres de los clientes y la información del pedido correspondiente solo para aquellos clientes que han realizado pedidos, podríamos utilizar la siguiente consulta SQL:

 
           SELECT Clientes.Nombre, Pedidos.Fecha, Pedidos.Total
           FROM Clientes
           INNER JOIN Pedidos
           ON Clientes.IDCliente = Pedidos.IDCliente;

En este ejemplo, la consulta SQL unirá las tablas "Clientes" y "Pedidos" utilizando la columna "IDCliente" . Solo se mostrarán los registros para los cuales existe una coincidencia en ambas tablas.

LEFT JOIN:

La cláusula LEFT JOIN se utiliza para mostrar todos los registros de la tabla izquierda (primera tabla especificada en la consulta SQL) y los registros coincidentes de la tabla derecha (segunda tabla especificada en la consulta SQL). Por ejemplo, si queremos mostrar los nombres de todos los clientes y la información del pedido correspondiente, incluso si el cliente no ha realizado ningún pedido, podríamos utilizar la siguiente consulta SQL:

           SELECT Clientes.Nombre, Pedidos.Fecha, Pedidos.Total
           FROM Clientes
           LEFT JOIN Pedidos
           ON Clientes.IDCliente = Pedidos.IDCliente;

En este ejemplo, la consulta SQL unirá las tablas "Clientes" y "Pedidos" utilizando la columna "IDCliente" . Se mostrarán todos los registros de la tabla "Clientes" y los registros coincidentes de la tabla "Pedidos" . Si un cliente no ha realizado ningún pedido, la información del pedido será nula.

RIGHT JOIN:

La cláusula RIGHT JOIN se utiliza para mostrar todos los registros de la tabla derecha (segunda tabla especificada en la consulta SQL) y los registros coincidentes de la tabla izquierda (primera tabla especificada en la consulta SQL). Por ejemplo, si queremos mostrar la información de todos los pedidos y el nombre del cliente correspondiente, incluso si el cliente no existe en la tabla "Clientes" , podríamos utilizar la siguiente consulta SQL:

           SELECT Clientes.Nombre, Pedidos.Fecha, Pedidos.Total
           FROM Clientes
           RIGHT JOIN Pedidos
           ON Clientes.IDCliente = Pedidos.IDCliente;

En este ejemplo, la consulta SQL unirá las tablas "Clientes" y "Pedidos" utilizando la columna "IDCliente" . Se mostrarán todos los registros de la tabla "Pedidos" y los registros coincidentes de la tabla "Clientes" . Si un pedido no tiene un cliente correspondiente en la tabla "Clientes" , el nombre del cliente será nulo.

Importancia de aprender SQL en la actualidad

SQL es un lenguaje de programación que se utiliza para administrar y manipular datos en las bases de datos relacionales. Hoy en día, la mayoría de las empresas y organizaciones manejan grandes cantidades de datos y utilizan bases de datos relacionales para almacenar y acceder a esa información.

Aprender SQL es importante porque permite a los profesionales trabajar con datos de manera más eficiente y efectiva. Con SQL , se pueden realizar consultas y búsquedas complejas en grandes conjuntos de datos, lo que facilita la identificación de patrones y tendencias. También permite la creación y administración de bases de datos, lo que es fundamental para el funcionamiento de muchas aplicaciones y sistemas empresariales.

Además, el conocimiento de SQL es una habilidad altamente valorada en el mercado laboral actual, especialmente en roles relacionados con análisis de datos, desarrollo de software y administración de bases de datos. Saber SQL puede abrir oportunidades laborales en una amplia variedad de industrias y puede ser una habilidad altamente transferible a medida que los profesionales avanzan en sus carreras.
